1. Control de Calidad Genómico (vcf-filter)

📌 Introducción: La secuenciación masiva (NGS) genera errores técnicos. Analizar datos "sucios" puede llevar a conclusiones falsas sobre la relación entre plantas. El filtrado es la etapa más crítica para asegurar que solo la variación biológica real sea considerada.
💻 Explicación Bioinformática: El software escanea cada línea del archivo VCF (un SNP). Para cada uno, evalúa los campos de calidad (QUAL) y la representatividad en la población. Si un SNP solo aparece en una planta o tiene demasiados valores nulos (.), el algoritmo lo descarta de la memoria para reducir el ruido estadístico.
\[ \text{MAF} = \frac{\min(\sum f_{ref}, \sum f_{alt})}{\sum (f_{ref} + f_{alt})} \]
java -jar biocenicana.jar vcf-filter -v raw.vcf -o clean.vcf --min-maf 0.05
💡 Analogía Simple: Es como limpiar granos de café. Antes de tostarlos, quitas las piedras y los granos dañados. Solo el café de calidad premium (SNPs confiables) pasa al siguiente paso.

2. Estructura Poblacional (pop-structure)

📌 Introducción: En fitomejoramiento, es vital saber si los individuos están emparentados por origen geográfico o linaje. Esto evita la endogamia y ayuda a seleccionar padres divergentes para maximizar el vigor híbrido.
💻 Explicación Bioinformática: Convertimos las letras del ADN (A, T, C, G) en números (dosis alélica). Estos números forman una matriz gigante. Aplicamos una técnica llamada Descomposición en Valores Singulares (SVD) para encontrar los ejes de mayor variación, permitiendo ver en 2D lo que antes eran miles de columnas de datos.
\[ Z = U \Sigma V^T \]
java -jar biocenicana.jar pop-structure -v clean.vcf -p 10 -o reporte
💡 Analogía Simple: Es como tomar una fotografía de una multitud desde un drone. Desde el suelo todos parecen iguales, pero desde arriba puedes ver claramente que hay diferentes grupos o "islas" de personas.

3. Desequilibrio de Ligamiento (ld-decay)

📌 Introducción: El LD nos dice qué tan grande es la "unidad de herencia". Si el LD cae rápido, necesitamos muchos marcadores para encontrar un gen de interés. Si cae lento, unos pocos marcadores cubren mucho terreno. Es el mapa de resolución del genoma.
💻 Explicación Bioinformática: El algoritmo mide la correlación estadística (\(r^2\)) entre pares de SNPs cercanos en el mismo cromosoma. Al graficar esta correlación versus la distancia física (en pares de bases), obtenemos una curva que nos indica la resolución necesaria para estudios de GWAS en esa población.
\[ r^2 = \frac{(P_{AB} - P_A P_B)^2}{P_A(1-P_A)P_B(1-P_B)} \]
java -jar biocenicana.jar ld-decay -v clean.vcf -o ld_report
💡 Analogía Simple: Es como la señal de Wi-Fi. Cerca del router (un gen), la señal es fuerte y constante. A medida que te alejas por el pasillo (el cromosoma), la señal se pierde. El LD mide qué tan lejos llega esa señal genética.

4. Relación Genómica (Kinship)

📌 Introducción: El parentesco tradicional (pedigrí) suele tener errores. El Kinship genómico mide la verdadera proporción de ADN compartido. Es la base de la Selección Genómica moderna para predecir el rendimiento de una planta antes de que crezca.
💻 Explicación Bioinformática: Calculamos una matriz de relación genómica (GRM) usando el método de VanRaden. Esta matriz compara cada individuo contra todos los demás, pesando cada SNP por su importancia en la población (MAF), generando una diagonal de auto-parentesco y valores de covarianza para pares de individuos.
\[ \mathbf{K} = \frac{\mathbf{Z} \mathbf{Z}^T}{\text{constante de escala}} \]
💡 Analogía Simple: Es como un test de ADN de paternidad, pero para miles de plantas a la vez. Nos dice exactamente qué tan "primos" o "hermanos" son dos individuos sin importar lo que digan los registros de campo.
